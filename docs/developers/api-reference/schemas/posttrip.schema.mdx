The enriched, read-time view of a `TRIP` post's `typeData` — computed
fresh on every read, never stored as-is. A trip's check-ins are its
`Comment` rows with `metadata.kind: "trip_checkin"` (see `Comment`
below); this object aggregates them rather than requiring a second
fetch.
