# Original project brief

> **Historical document.** This is the original kickoff brief describing how Famlin was meant to be built from scratch, step by step. The project is now implemented, so this is kept for context and intent — it is **not** the source of truth for how the code currently works. For day-to-day operational guidance (commands, conventions, architecture as-built), see the [Architecture](./architecture) page.

The brief covered the original step-by-step plan: repo structure, backend setup, auth, the admin UI, notifications, the Expo app, Docker Compose deployment, and EAS builds.
